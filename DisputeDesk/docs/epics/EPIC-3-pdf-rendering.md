# EPIC 3 — PDF Rendering and Storage

> **Status:** Pending
> **Week:** 3
> **Dependencies:** EPIC 2

## Goal

Generate a professional, print-ready PDF evidence pack and make it downloadable via signed URL.

## Tasks

### 3.1 — PDF Template Design
- Use `@react-pdf/renderer` for deterministic server-side PDF generation.
- Template sections:
  - **Cover page**: Shop name, dispute reference, generation date, completeness score.
  - **Dispute summary**: Reason, amount, dates, status.
  - **Order summary**: Line items table, amounts, customer info (redacted).
  - **Shipping/tracking proof**: Carrier, tracking number, URL, delivery status, ship date.
  - **Policy snapshots**: Refund/shipping/terms text (truncated if long).
  - **Manual attachments list**: File names, types, upload dates (referenced, not embedded).
  - **Audit trail excerpt**: Key events with timestamps.
  - **Footer**: "Generated by Dispute Ops — evidence for internal use and dispute response."

### 3.2 — Render Pipeline
- `lib/packs/renderPdf.tsx` — accepts pack JSON, returns PDF buffer.
- Runs as async `render_pdf` job:
  1. Load pack from DB.
  2. Call `renderPdf()`.
  3. Upload to Supabase Storage at `{shopId}/{packId}/{timestamp}.pdf`.
  4. Update `evidence_packs.pdf_path`.
  5. Insert `audit_events` (`event_type: pdf_rendered`).

### 3.3 — Download via Signed URL
- `GET /api/packs/:packId/download` — 1-hour signed URL from Supabase Storage.
- "Download PDF" button on pack preview opens in new tab.

### 3.4 — Regeneration Behavior
- New PDF rendered with new timestamp path.
- Old path kept in audit log; `pdf_path` updated.
- Audit event: `{ previous_path, new_path }`.

### 3.5 — Supabase Storage Buckets
- `evidence-packs` bucket (private, no public access).
- `evidence-uploads` bucket for manual uploads (private).
- Storage policies: service role writes only; signed URLs for read.

## Key Files
- `lib/packs/renderPdf.tsx`
- `app/api/packs/[packId]/render-pdf/route.ts`
- `app/api/packs/[packId]/download/route.ts`

## Acceptance Criteria
- [ ] PDF includes all template sections with correct data.
- [ ] PDF generation success rate > 95% (measured over 20+ packs).
- [ ] PDF stored in Supabase Storage; `pdf_path` set on pack row.
- [ ] Signed URL download works and expires after 1 hour.
- [ ] Regeneration creates new version; audit log records both paths.
- [ ] PDF file size < 5 MB for typical pack.
