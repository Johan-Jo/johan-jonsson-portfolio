# EPIC 3 тАФ PDF Rendering and Storage

> **Status:** Done
> **Week:** 3
> **Dependencies:** EPIC 2

## Goal

Generate a professional, print-ready PDF evidence pack and make it downloadable via signed URL.

## Implementation

### 3.1 тАФ PDF Template (`lib/packs/pdf/`)

- **`styles.ts`** тАФ `@react-pdf/renderer` StyleSheet with project-branded tokens (cover page, sections, tables, key-value rows, checklist, audit trail, footer).
- **`EvidencePackDocument.tsx`** тАФ React-PDF document component with two pages:
  - **Cover page**: Shop name, dispute reference, generation date, completeness score with color coding (green тЙе80, amber тЙе50, red <50), blocker count.
  - **Content page**:
    - **Evidence Checklist** тАФ field-level presence with тЬУ/тЬЧ markers and "REQUIRED" labels for missing required fields.
    - **Blockers / Recommended Actions** тАФ color-coded banner boxes.
    - **Order Details** тАФ order name, dates, financial/fulfillment status, totals table, line items table.
    - **Shipping & Tracking** тАФ per-fulfillment: shipped/delivered dates, carrier, tracking number/URL.
    - **Store Policies** тАФ refund/shipping/terms snapshots with truncated previews and character counts.
    - **Manual Attachments** тАФ table listing uploaded file names, types, and upload dates.
    - **Audit Trail** тАФ timestamped event log (last 50 events).
  - **Footer** (fixed): "Generated by DisputeDesk" with pack ID prefix and generation timestamp.

### 3.2 тАФ Render Pipeline

- **`lib/packs/renderPdf.tsx`** тАФ Dynamic-imports `@react-pdf/renderer` and the document component at runtime (avoids webpack bundling native `yoga-layout` dep at build time). Calls `renderToBuffer()` and returns a Node `Buffer`.
- **`lib/jobs/handlers/renderPdfJob.ts`** тАФ Full `render_pdf` job handler:
  1. Loads pack row + dispute + shop + audit events from Supabase.
  2. Assembles `PackPdfData` from `pack_json.sections`, checklist, blockers, recommended actions.
  3. Calls `renderPackPdf()`.
  4. Uploads PDF buffer to Supabase Storage bucket `evidence-packs` at `{shopId}/{packId}/{timestamp}.pdf`.
  5. Updates `evidence_packs.pdf_path`.
  6. Logs `pdf_rendered` audit event with `storagePath`, `previousPath`, and `fileSizeBytes`.
  7. On failure logs `job_failed` audit event and re-throws.
- Dispatched via existing job worker (`app/api/jobs/worker/route.ts`).

### 3.3 тАФ Render Trigger

- **`POST /api/packs/:packId/render-pdf`** тАФ Enqueues a `render_pdf` job, logs `job_queued` audit event, returns `{ jobId, status: "queued" }` with 202.

### 3.4 тАФ Download via Signed URL

- **`GET /api/packs/:packId/download`** тАФ Returns a 1-hour signed URL from Supabase Storage for the pack's `pdf_path`. Returns 404 if no PDF has been rendered yet.

### 3.5 тАФ Regeneration Behavior

- Each render creates a new timestamp-based path; old path is preserved in audit log `previousPath`.
- `pdf_path` on the pack row always points to the latest version.

### 3.6 тАФ Dynamic Import Pattern (Estimate Pro standard)

- **`lib/packs/pdf/reactPdfRuntime.ts`** тАФ Dedicated module exporting `getReactPdfRenderer()` and `getEvidencePackDocumentModule()` as async dynamic imports. Same pattern as Estimate Pro's `src/lib/pdf/estimate/reactPdfRuntime.ts`.
- **`lib/packs/renderPdf.tsx`** тАФ Uses the runtime module to keep `@react-pdf/renderer` out of webpack's static analysis. No `serverExternalPackages` needed.
- **`export const runtime = "nodejs"`** set on render-pdf, download, and worker API routes.

### 3.7 тАФ UI Integration

**Embedded App (`app/(embedded)/app/packs/[packId]/page.tsx`)**:
- "PDF Export" card with:
  - "Render PDF" / "Re-render PDF" button (shows when no render is in progress).
  - "Download PDF" button (appears when `pdf_path` is set).
  - "Rendering PDF..." banner with auto-refresh while `active_pdf_job` is present.
  - Last-rendered timestamp display.

**External Portal (`app/(portal)/portal/packs/[packId]/page.tsx`)**:
- "PDF Export" section with identical functionality using portal design system.

Both UIs poll for updates during rendering and stop polling when complete.
